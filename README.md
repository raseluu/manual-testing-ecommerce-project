# Manual Testing Project – E-Commerce Website

## Project Overview

This project demonstrates manual testing of an e-commerce web application. The testing was conducted on the AutomationExercise website to verify core functionalities such as user registration, login, product browsing, product search, cart operations, and contact form submission.

The goal of this project is to simulate a real-world QA workflow by preparing test documentation, executing test cases, and reporting defects.

## Website Under Test

https://automationexercise.com

## Testing Approach

Manual testing was performed to validate the functionality and usability of the website. The testing focused on key user workflows and verifying that the application behaves as expected.

### Testing Types

* Functional Testing
* UI Testing
* Usability Testing

## Modules Tested

The following modules were tested during this project:

* Homepage
* Navigation Menu
* User Registration
* Login / Logout
* Product Listing
* Product Details
* Product Search
* Cart Functionality
* Contact Form

## Bugs Identified

The following issues were discovered during testing:

1. **Search Functionality Issue**
   No message is displayed when the product search returns no results.

2. **Product Alignment Issue**
   Product cards appear slightly misaligned on the product listing page.

Detailed information about these issues can be found in the Bug Report document.
