# Test Plan

## Project Title

Manual Testing of AutomationExercise E-Commerce Website

## Objective

The objective of this testing project is to perform manual testing on the AutomationExercise website to verify that key functionalities of the e-commerce platform work as expected. The testing focuses on validating core user flows such as authentication, product browsing, search, cart operations, and contact form submission.

## Website Under Test

https://automationexercise.com

## Testing Type

The following testing types were performed:

* Functional Testing
* UI Testing
* Usability Testing

## Scope of Testing

The following modules were included in testing:

* Homepage
* Navigation Menu
* User Registration
* Login / Logout
* Product Listing
* Product Details
* Product Search
* Cart Functionality
* Contact Form

## Out of Scope

The following areas were not included in this testing project:

* Payment gateway testing
* Performance testing
* Security testing
* Mobile device testing
* Cross-browser compatibility testing

## Identified Issues

Two issues were identified during testing:

1. No message displayed when product search returns no results.
2. Product cards appear misaligned on the product listing page.

## Conclusion

The AutomationExercise website generally performs well for the tested functionalities. Most core features such as registration, login, product browsing, cart operations, and contact form submission worked as expected. However, minor usability and UI issues were identified and documented in the bug report.
